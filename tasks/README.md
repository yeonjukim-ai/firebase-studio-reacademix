# 작업 목록 (Tasks)

이 폴더는 ReAcademix 프로젝트의 향후 개선 작업들을 우선순위별로 정리한 문서입니다.

## 📁 폴더 구조

```
tasks/
├── high-priority/      # 🔴 우선순위 높음 (1-2개월)
├── medium-priority/    # 🟡 우선순위 중간 (2-4개월)
├── low-priority/       # 🟢 우선순위 낮음 (4개월 이후)
└── README.md          # 이 파일
```

## 📊 우선순위별 작업 요약

### 🔴 우선순위 높음 (High Priority)
예상 총 시간: 4-6주

1. **테스트 코드 추가** (2-3주)
   - Unit 테스트 및 Integration 테스트 작성
   - 목표 커버리지: 80% 이상

2. **에러 처리 강화** (1주)
   - Error Boundary 구현
   - 통일된 에러 처리 유틸리티
   - 에러 로깅 시스템

3. **접근성 개선** (1-2주)
   - ARIA 라벨 추가
   - 키보드 네비게이션
   - 포커스 관리

### 🟡 우선순위 중간 (Medium Priority)
예상 총 시간: 7-9주

4. **컴포넌트 분리 및 리팩토링** (2주)
   - 대형 컴포넌트를 작은 컴포넌트로 분리

5. **상태 관리 개선** (1주)
   - Zustand 또는 Jotai 도입
   - 전역 상태 관리

6. **API 호출 최적화** (2주)
   - React Query 또는 SWR 도입
   - 서버 상태 관리 및 캐싱

7. **Firebase Firestore 연동** (2-3주)
   - 더미 데이터를 실제 데이터베이스로 교체
   - 실시간 업데이트 구현

### 🟢 우선순위 낮음 (Low Priority)
예상 총 시간: 12-15주

8. **성능 최적화** (2-3주)
9. **검색 기능 강화** (1주)
10. **필터 기능 개선** (1주)
11. **진행 중 작업 취소 기능** (1주)
12. **키보드 단축키** (1주)
13. **타입 안정성 강화** (1주)
14. **리포트 기능 확장** (3-4주)
15. **대시보드 개선** (2-3주)

## 📝 작업 진행 방법

1. 각 작업 파일을 열어 상세 작업 내용 확인
2. 작업 시작 전 관련 이슈 생성 (GitHub)
3. 작업 완료 후 체크리스트 업데이트
4. 완료된 작업은 해당 파일에 ✅ 표시

## ✅ 완료된 작업

완료된 작업은 각 파일의 체크리스트에서 확인할 수 있습니다.

## 📌 참고 사항

- 각 작업의 예상 시간은 참고용이며, 실제 소요 시간은 다를 수 있습니다.
- 작업 순서는 우선순위를 따르되, 의존성이 있는 작업은 순서를 조정할 수 있습니다.
- 새로운 작업 아이디어가 생기면 적절한 우선순위 폴더에 추가하세요.
