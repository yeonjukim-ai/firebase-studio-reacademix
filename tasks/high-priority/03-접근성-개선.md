# 접근성 개선

## 우선순위: 🔴 높음
## 예상 시간: 1-2주

## 작업 내용

### 1. ARIA 라벨 추가
- [ ] 모든 인터랙티브 요소에 ARIA 라벨 추가
  - [ ] 버튼에 `aria-label` 추가
  - [ ] 입력 필드에 `aria-label` 또는 `aria-labelledby` 추가
  - [ ] 테이블에 `aria-label` 추가
  - [ ] 차트에 `aria-label` 추가
- [ ] 폼 요소에 `aria-describedby` 추가 (도움말 텍스트)
- [ ] 상태 변경 시 `aria-live` 영역 활용

### 2. 키보드 네비게이션
- [ ] 모든 인터랙티브 요소 키보드 접근 가능 확인
  - [ ] 버튼, 링크, 입력 필드
  - [ ] 드롭다운 메뉴
  - [ ] 모달 다이얼로그
  - [ ] 테이블 행 선택
- [ ] Tab 순서 논리적으로 구성
- [ ] 포커스 표시 명확히 (focus-visible 스타일)

### 3. 포커스 관리
- [ ] 모달 열릴 때 포커스 트랩 구현
  - [ ] `Dialog` 컴포넌트 포커스 트랩 확인
  - [ ] 모달 내부에서만 Tab 이동 가능
  - [ ] 모달 닫을 때 원래 포커스 위치로 복귀
- [ ] 드롭다운 메뉴 포커스 관리
- [ ] 동적 콘텐츠 추가 시 포커스 이동

### 4. 스크린 리더 지원
- [ ] 의미론적 HTML 사용 확인
  - [ ] `<nav>`, `<main>`, `<header>`, `<footer>` 등 적절히 사용
  - [ ] 제목 계층 구조 (`<h1>` ~ `<h6>`) 확인
- [ ] 이미지에 대체 텍스트 추가 (필요한 경우)
- [ ] 아이콘 버튼에 텍스트 라벨 또는 aria-label 추가

### 5. 색상 대비 확인
- [ ] WCAG AA 기준 충족 확인
  - [ ] 텍스트와 배경 색상 대비 4.5:1 이상
  - [ ] 대형 텍스트는 3:1 이상
- [ ] 색상만으로 정보 전달하지 않기 (아이콘, 텍스트 병행)

### 6. 접근성 테스트
- [ ] 키보드만으로 모든 기능 사용 가능한지 테스트
- [ ] 스크린 리더로 테스트 (NVDA, JAWS 등)
- [ ] 접근성 검사 도구 사용 (axe DevTools, WAVE 등)

## 참고 사항
- Radix UI 컴포넌트는 기본적으로 접근성을 고려하여 설계됨
- WCAG 2.1 AA 기준 준수 목표
- shadcn/ui 컴포넌트의 접근성 기능 최대한 활용
