## 전체 사용자 여정

### 1. 초기 접근 및 인증

```
사용자 접속
  ↓
로그인 페이지 (/login)
  ├── 이메일 입력
  ├── 비밀번호 입력
  └── 로그인 버튼 클릭
  ↓
인증 처리 (Firebase Authentication)
  ↓
대시보드로 리다이렉트 (/dashboard)
```

**주요 화면**:
- 로그인 페이지: 깔끔한 카드 레이아웃, 로고, 입력 필드, 비밀번호 찾기 링크

**사용자 목표**: 시스템에 안전하게 접근

---

### 2. 대시보드 탐색

```
대시보드 진입
  ↓
KPI 카드 확인 (4개)
  ├── 총 학생 수
  ├── 활성 학생 수
  ├── 이번 달 리포트 생성 수
  └── 평균 출석률
  ↓
성과 차트 확인 (지점별 성과 추이)
  ↓
최근 활동 테이블 확인
  ├── 최근 생성된 리포트 목록
  └── "전체 보기" 클릭 → 리포트 페이지로 이동
```

**주요 화면**:
- 대시보드 페이지: 그리드 레이아웃, KPI 카드, 차트, 테이블
- 상단 헤더: 검색, 알림, 사용자 프로필
- 좌측 사이드바: 네비게이션 메뉴

**사용자 목표**: 전체 현황을 빠르게 파악

**액션 버튼**:
- "데이터 업로드" → 데이터 관리 페이지로 이동
- "리포트 생성" → 리포트 페이지로 이동

## 핵심 워크플로우

### 워크플로우 1: 학생 데이터 관리

#### 1.1 학생 목록 조회

```
학생 관리 페이지 접근 (/dashboard/students)
  ↓
학생 목록 테이블 표시
  ├── 이름, 지점, 반, 상태, 등록일, 최종 리포트일
  └── 각 행에 "상세 보기" 버튼
```

#### 1.2 학생 검색

```
검색 입력 필드에 키워드 입력
  ├── 학생 이름
  ├── 반 이름
  └── 학생 ID
  ↓
실시간 필터링 (debounce 없음, 즉시 반영)
  ↓
필터링된 결과 표시
```

**사용자 경험**:
- 검색어 입력 즉시 결과 업데이트
- 검색어 하이라이트 없음 (향후 개선 가능)

#### 1.3 필터링

```
"필터" 버튼 클릭
  ↓
드롭다운 메뉴 열림
  ├── 학생 상태 필터
  │   ├── active
  │   ├── inactive
  │   └── on_leave
  └── 지점 필터
      ├── 지점 A
      ├── 지점 B
      └── ...
  ↓
체크박스 선택/해제
  ↓
필터링된 결과 즉시 반영
```

**사용자 경험**:
- 다중 선택 가능
- 필터 조합 가능 (상태 + 지점)
- 필터 초기화 기능 없음 (향후 개선 가능)

#### 1.4 학생 상세 정보 조회

```
"상세 보기" 버튼 클릭
  ↓
Dialog 모달 열림
  ├── 학생 이름 (제목)
  ├── ID
  ├── 지점
  ├── 반
  ├── 평균 점수
  └── 출석률
  ↓
Dialog 닫기 (X 버튼 또는 배경 클릭)
```

**사용자 경험**:
- 모달 오버레이로 포커스 유지
- ESC 키로 닫기 가능 (기본 동작)

### 워크플로우 2: 리포트 생성

#### 2.1 리포트 생성 시작

```
리포트 관리 페이지 접근 (/dashboard/reports)
  ↓
"리포트 생성" 탭 선택 (기본값)
  ↓
리포트 생성 폼 표시
  ├── 리포트 템플릿 선택
  │   ├── 월간 성과 리포트
  │   ├── 시험 분석 리포트
  │   └── 출결 리포트
  ├── 리포트 대상 선택
  │   ├── 전체 학생
  │   ├── A반
  │   ├── B반
  │   └── 개별 선택
  └── 리포트 기간 선택
      └── 날짜 범위 선택 (Calendar)
  ↓
"리포트 생성 시작" 버튼 클릭
```

**사용자 경험**:
- 날짜 선택은 Popover로 Calendar 컴포넌트 표시
- 2개월 캘린더 동시 표시
- 날짜 범위 선택 가능

#### 2.2 리포트 생성 진행

```
생성 시작 버튼 클릭
  ↓
생성 중 상태 표시
  ├── 로딩 스피너
  ├── 진행률 표시 (Progress Bar)
  └── 진행률 퍼센트 텍스트
  ↓
자동 진행률 업데이트 (500ms 간격)
  ├── 0% → 90%까지 랜덤 증가
  └── 5초 후 100% 도달
  ↓
생성 완료 또는 실패 처리
```

**사용자 경험**:
- 진행률이 시각적으로 표시됨
- 취소 기능 없음 (향후 개선 가능)
- 생성 중 다른 작업 불가

#### 2.3 리포트 생성 완료

```
생성 완료 (80% 확률)
  ↓
성공 Alert 표시
  ├── "리포트 생성 완료!" 제목
  └── 다운로드 안내 메시지
  ↓
하단 버튼 표시
  ├── "새 리포트 생성" (outline)
  └── "다운로드" (primary)
  ↓
"다운로드" 클릭 → 파일 다운로드 (향후 구현)
"새 리포트 생성" 클릭 → 폼 초기화
```

#### 2.4 리포트 생성 실패

```
생성 실패 (20% 확률)
  ↓
에러 Alert 표시
  ├── "생성 실패" 제목
  └── 오류 메시지
  ↓
"다시 시도" 버튼 표시
  ↓
"다시 시도" 클릭 → 폼 초기화 및 재시도
```

### 워크플로우 3: 데이터 업로드 및 검증

#### 3.1 파일 업로드

```
데이터 관리 페이지 접근 (/dashboard/data)
  ↓
파일 업로드 영역 표시
  ├── 드래그 앤 드롭 영역
  └── "드래그 앤 드롭 또는 클릭하여 파일 업로드" 안내
  ↓
파일 선택 방법
  ├── 방법 1: 드래그 앤 드롭
  │   └── 파일을 영역에 드롭
  └── 방법 2: 클릭
      └── 파일 선택 다이얼로그 열림
  ↓
파일 선택 완료
  ├── CSV 파일만 허용
  └── 파일 형식 검증
```

**사용자 경험**:
- 드래그 앤 드롭 시각적 피드백 (호버 효과)
- 파일 형식 오류 시 토스트 알림
- 파일 크기 제한 없음 (향후 개선 가능)

#### 3.2 파일 확인 및 검증 시작

```
파일 선택 완료
  ↓
파일 정보 카드 표시
  ├── 파일 아이콘
  ├── 파일 이름
  ├── 파일 크기 (KB)
  └── 삭제 버튼 (X)
  ↓
"업로드 및 데이터 검증" 버튼 표시
  ↓
버튼 클릭
  ↓
검증 중 상태 표시
  ├── 로딩 스피너
  ├── "AI가 데이터를 검증하고 있습니다..." 메시지
  └── "잠시만 기다려주세요." 안내
  ↓
AI 검증 API 호출 (2초 시뮬레이션)
```

**사용자 경험**:
- 검증 중 취소 불가
- 검증 시간이 길 경우 사용자 대기

#### 3.3 검증 결과 확인

**시나리오 A: 검증 성공**

```
검증 완료 (isValid: true)
  ↓
성공 Alert 표시
  ├── "검증 완료" 제목
  └── "모든 데이터가 성공적으로 시스템에 반영되었습니다." 메시지
  ↓
뒤로가기 버튼으로 초기 상태로 복귀
```

**시나리오 B: 검증 실패 (에러 발견)**

```
검증 완료 (isValid: false)
  ↓
에러 Alert 표시
  ├── "검증 오류 발견" 제목
  └── "N개의 오류가 발견되었습니다." 메시지
  ↓
수정 가능한 테이블 표시
  ├── CSV 헤더 행
  ├── 데이터 행들
  ├── 에러가 있는 셀: 빨간색 배경
  └── 각 셀에 Input 필드
  ↓
에러 메시지 표시
  └── 각 에러 셀 아래에 에러 설명
```

#### 3.4 데이터 수정 및 저장

```
에러 셀 확인
  ↓
에러 메시지 읽기
  ↓
Input 필드에 올바른 값 입력
  ↓
다른 셀도 수정 (필요 시)
  ↓
"수정 내용 저장" 버튼 클릭
  ↓
저장 완료 토스트 알림
  └── "수정된 데이터가 성공적으로 저장되었습니다."
  ↓
초기 상태로 복귀
```

**사용자 경험**:
- 실시간 입력 가능
- 에러 셀은 시각적으로 강조
- 저장 전 검증 없음 (향후 개선 가능)

## 상세 사용자 시나리오

### 시나리오 1: 월간 리포트 생성

**사용자**: 학원 관리자  
**목표**: 이번 달 전체 학생의 월간 성과 리포트 생성

**단계**:
1. 대시보드에서 "리포트 생성" 버튼 클릭
2. 리포트 관리 페이지에서 "리포트 생성" 탭 확인
3. 리포트 템플릿: "월간 성과 리포트" 선택
4. 리포트 대상: "전체 학생" 선택
5. 리포트 기간: 이번 달 1일 ~ 마지막일 선택
6. "리포트 생성 시작" 버튼 클릭
7. 생성 진행률 확인 (약 5초)
8. 생성 완료 후 "다운로드" 버튼 클릭
9. 리포트 파일 다운로드

**예상 소요 시간**: 약 1분

---

### 시나리오 2: 신규 학생 데이터 업로드

**사용자**: 학원 관리자  
**목표**: 신규 입학생 데이터를 CSV로 업로드하고 검증

**단계**:
1. 대시보드에서 "데이터 업로드" 버튼 클릭
2. 데이터 관리 페이지에서 CSV 파일 드래그 앤 드롭
3. 파일 정보 확인
4. "업로드 및 데이터 검증" 버튼 클릭
5. AI 검증 진행 대기 (약 2초)
6. 검증 결과 확인
   - 성공: 완료 메시지 확인
   - 실패: 에러 셀 확인 및 수정
7. 수정 완료 후 "수정 내용 저장" 클릭
8. 저장 완료 확인

**예상 소요 시간**: 약 3-5분 (에러 수정 포함)

---

### 시나리오 3: 특정 학생 정보 조회

**사용자**: 학원 관리자  
**목표**: 특정 학생의 상세 정보 확인

**단계**:
1. 사이드바에서 "학생 관리" 클릭
2. 검색 필드에 학생 이름 입력
3. 검색 결과에서 해당 학생 찾기
4. "상세 보기" 버튼 클릭
5. Dialog에서 상세 정보 확인
   - ID, 지점, 반, 평균 점수, 출석률
6. Dialog 닫기

**예상 소요 시간**: 약 30초

## 사용자 인터랙션 패턴

### 1. 네비게이션 패턴

**주요 네비게이션 경로**:
- 사이드바 메뉴: 페이지 간 이동
- 상단 헤더 검색: 전역 검색 (향후 구현)
- 액션 버튼: 특정 작업으로 이동
  - "데이터 업로드" → 데이터 관리 페이지
  - "리포트 생성" → 리포트 페이지

**브레드크럼**:
- 상단 헤더에 "ReAcademix / Dashboard" 표시
- 현재 페이지 경로 표시

---

### 2. 피드백 패턴

**로딩 상태**:
- 스피너 애니메이션
- 진행률 바 (리포트 생성)
- 로딩 메시지
