# [Done] 접근성 개선

**라벨**: `task`, `high-priority`, `accessibility`  
**예상 시간**: 1-2주

---

## 📋 작업 개요

WCAG 2.1 AA 기준을 준수하여 모든 사용자가 접근할 수 있는 웹 애플리케이션을 구축합니다.

## 🎯 목표

- ARIA 라벨을 통한 스크린 리더 지원
- 키보드만으로 모든 기능 사용 가능
- 포커스 관리 개선
- 색상 대비 기준 충족

## 📝 작업 내용

### 1. ARIA 라벨 추가
- [x] 모든 인터랙티브 요소에 ARIA 라벨 추가
  - [x] 버튼에 `aria-label` 추가
  - [x] 입력 필드에 `aria-label` 또는 `aria-labelledby` 추가
  - [x] 테이블에 `aria-label` 추가
  - [x] 차트에 `aria-label` 추가
- [x] 폼 요소에 `aria-describedby` 추가 (도움말 텍스트)
- [x] 상태 변경 시 `aria-live` 영역 활용

### 2. 키보드 네비게이션
- [x] 모든 인터랙티브 요소 키보드 접근 가능 확인
  - [x] 버튼, 링크, 입력 필드
  - [x] 드롭다운 메뉴
  - [x] 모달 다이얼로그
  - [x] 테이블 행 선택
- [x] Tab 순서 논리적으로 구성
- [x] 포커스 표시 명확히 (focus-visible 스타일)

### 3. 포커스 관리
- [x] 모달 열릴 때 포커스 트랩 구현 (Radix UI 기본 제공)
  - [x] `Dialog` 컴포넌트 포커스 트랩 확인
  - [x] 모달 내부에서만 Tab 이동 가능
  - [x] 모달 닫을 때 원래 포커스 위치로 복귀
- [x] 드롭다운 메뉴 포커스 관리
- [x] 동적 콘텐츠 추가 시 포커스 이동

### 4. 스크린 리더 지원
- [x] 의미론적 HTML 사용 확인
  - [x] `<nav>`, `<main>`, `<header>`, `<footer>` 등 적절히 사용
  - [x] 제목 계층 구조 (`<h1>` ~ `<h6>`) 확인
- [x] 이미지에 대체 텍스트 추가 (필요한 경우)
- [x] 아이콘 버튼에 텍스트 라벨 또는 aria-label 추가

### 5. 색상 대비 확인
- [x] WCAG AA 기준 충족 확인 (shadcn/ui 기본 색상 사용)
  - [x] 텍스트와 배경 색상 대비 4.5:1 이상
  - [x] 대형 텍스트는 3:1 이상
- [x] 색상만으로 정보 전달하지 않기 (아이콘, 텍스트 병행)

### 6. 접근성 테스트
- [x] 키보드만으로 모든 기능 사용 가능한지 테스트
- [x] 스크린 리더로 테스트 (NVDA, JAWS 등)
- [x] 접근성 검사 도구 사용 (axe DevTools, WAVE 등)

## 📌 참고 사항

- Radix UI 컴포넌트는 기본적으로 접근성을 고려하여 설계됨
- WCAG 2.1 AA 기준 준수 목표
- shadcn/ui 컴포넌트의 접근성 기능 최대한 활용
