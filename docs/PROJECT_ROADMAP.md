# ReAcademix 프로젝트 로드맵

이 문서는 GitHub 이슈를 기반으로 한 프로젝트 개발 일정 및 로드맵입니다.

**시작일**: 내일 (2024년 기준)  
**기준 이슈**: #4부터 시작

---

## 📅 전체 일정 개요

| 기간 | 마일스톤 | 주요 작업 |
|------|----------|----------|
| **1-2주** | Phase 1: 핵심 안정성 | 에러 처리, 접근성 개선 |
| **3-6주** | Phase 2: 코드 품질 개선 | 리팩토링, 상태 관리, API 최적화 |
| **7-10주** | Phase 3: 데이터베이스 연동 | Firestore 연동, 실시간 업데이트 |
| **11-14주** | Phase 4: 성능 및 기능 개선 | 성능 최적화, 검색/필터 강화 |
| **15주 이후** | Phase 5: 고급 기능 | 취소 기능, 단축키, 타입 안정성, 리포트 확장, 대시보드 개선 |

## 🗓️ 상세 일정표

### Phase 1: 핵심 안정성 (1-2주)

#### Week 1 (이슈 #4)
**에러 처리 강화** - 예상 시간: 1주
- Error Boundary 구현
- 통일된 에러 처리 유틸리티
- 에러 로깅 시스템
- 사용자 피드백 개선

**시작일**: 내일  
**완료 예정일**: 1주 후

---

#### Week 2 (이슈 #5)
**접근성 개선** - 예상 시간: 1-2주
- ARIA 라벨 추가
- 키보드 네비게이션
- 포커스 관리
- 스크린 리더 지원
- 색상 대비 확인
- 접근성 테스트

**시작일**: Week 1 완료 후  
**완료 예정일**: Week 2 종료 (또는 Week 3 초반)

## ⚠️ 주의사항

1. **예상 시간은 참고용**: 실제 소요 시간은 프로젝트 상황에 따라 달라질 수 있습니다.
2. **병렬 작업 가능**: 일부 이슈는 독립적이므로 병렬로 진행할 수 있습니다.
3. **의존성 확인**: 이슈 #9와 #17은 다른 작업 완료 후 시작하는 것이 좋습니다.
4. **우선순위 조정**: 프로젝트 상황에 따라 우선순위를 조정할 수 있습니다.
